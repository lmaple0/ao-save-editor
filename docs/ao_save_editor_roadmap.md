## 料理手册 ✅ 已完成 (2026-07-15)

- **偏移**: `0x19C98`、`0x19C9C`、`0x19CA0` 三份镜像 `u32` 位图。
- **格式**: bit 1~24 对应 24 道基础菜谱，bit 0 保留；全开掩码为 `0x01FFFFFE`。
- **写入安全**: 三份镜像同步更新，同时保留料理掩码以外的未知位。
- **名称映射**: 按 NISA/Falcom `t_cook._dt` 索引 1~24 映射到每组三件料理中的基础料理 ID，复用现有 `ao_item_i18n.json` 中英日名称。
- **GUI**: 新增料理手册标签页，每道菜一个复选框，支持全选/全不选和全局中文/英文/日文切换。
- **验证**: 120 份 Ao 存档的三镜像全部一致；9 个已知进度点的置位数与游戏内数量匹配；临时副本完成全开、保存、重新加载 roundtrip。

### 4. 怪物图鉴目录、诊断与逐条编辑 ✅ P0 深化完成 (2026-07-15)

- **偏移**: `0x1B370 ~ 0x1BCF7`，共 305 个固定槽 × 8 字节；`0x1BCF0` 是最后一条记录的起始地址。
- **记录格式**: `u32 code + u8 flag + u8 resistance + u8 stats + u8 get_item`。
- **NISA 完整资料判定**: `stats` 的低 7 位全部开启且 `get_item=0xFF`；击破计数与其他变体位可自然变化。继续兼容旧 Joyoland/BZH 的 `08 FE FF FF` payload。
- **参考目录**: `ao_monster_reference.json` 静态合并 `MONSTER_CODES`、Ouroboros/Falcom `t_dbmon.py` 身份层和本机 NISA 三语 `ms*.dat` 名称；305/305 覆盖，其中上游启用 283、注释待复核 22。
- **GUI**: 独立怪物图鉴页，支持按代码、CLE 中文/日文/NISA 英文怪物名、`ms*.dat`、等级、来源和地点搜索；名称与地点随全局语言切换；提供 75 个细分地点下拉筛选；显示未登记/部分资料/全资料；可多选设为全资料或未登记。
- **安全策略**: 未选择记录和未知代码原样保留；报告重复代码；容量不足则拒绝写入；保留快捷页一键全开。
- **本地化来源**: 三语怪物名由本机 `data_cn/battle/dat`、`data/battle/dat`、`data/battle_us/dat` 的 `ms*.dat` 直接读取；地点由场景 `MapIndex` 对照三个本机 `t_town._dt` 表。302 项直接来自场景扫描；`ms82004`、村正、十六夜三个 MapIndex 为 0 的事件战斗使用已知本地地点标签在本机地点表中解析。每条记录保留场景文件和来源类型。
- **可复现性**: `python -m tools.build_monster_reference --game-root <NISA游戏目录> --check` 检查生成文件是否与来源一致；`t_dbmon` 使用 AST/文本解析，不执行上游脚本。
- **待验证**: 游戏内逐条回读、22 条上游注释映射的实际可见性，以及 4 个 payload 字节的精细语义。

### 4.1 三语怪物详情数据与只读详情面板 ✅ P0.1 + P0.2 完成 (2026-07-15)

- **生成器**: `tools/build_monster_details.py` 独立实现带边界检查的 `ms*.dat` 只读解析，不加载旧 Decompiler 运行时，也不执行反汇编生成的 `.py`。
- **覆盖**: CLE 中文、Falcom 日文、NISA 英文各 333 个文件；名称 333/333、说明 316/333、自定义战技 319/333、有效动作脚本来源 333/333。
- **一致性**: 三语文件的完整数值结构逐文件一致；当前 305 个存档图鉴怪物全部能按 `ms` 文件关联详情，名称和等级与现有参考目录一致。
- **GUI**: 现有怪物图鉴新增只读详情面板，显示 HP/EP/CP、EXP、战斗属性、七属性有效率、抗性原始位图、耀晶片、掉落、战技、说明和 `ms/as` 来源。
- **保守处理**: `ms60000.dat`、`ms60001.dat` 存在上游解析器未消费的尾部扩展；只记录大小和 SHA-256，不解释、不写回。
- **验证**: 37 项单元测试通过，详情目录和 UI 不提供保存接口。

### 4.2 存档诊断/审计报告 ✅ P0 完成 (2026-07-15)

- **深模块**: `ao_save_audit.py` 通过单一 `audit()` 接口执行只读结构检查，并由 `render()` 输出中/英/日报告；GUI 不持有诊断规则。
- **覆盖**: 存档大小、当前校验和、难度、物品未知/重复/数量/孤立槽、队伍与外观 ID、料理位图镜像、怪物未知/重复/部分状态/孤立状态、参考数据版本。
- **安全策略**: 只报告，不自动修复；没有保存接口，不改变输入 `bytearray`。部分怪物状态只给提示，继续保持未知子位只读。
- **结构**: `ao_save_layout.py` 成为编辑器和审计器共享的唯一布局常量来源，避免偏移复制漂移；2.3 MB 怪物详情 JSON 改为首次查看时懒加载。
- **验证**: 47 项单元测试通过；两个压缩样例均为 0 错误/0 警告、哈希不变，诊断耗时约 5–6 ms；Windows Python 3.13 隐藏窗口验证 12 标签页和三语切换。

### 4.3 角色 / 怪物 / 战斗脚本交叉索引 ✅ P1.1–P1.4 只读阶段完成 (2026-07-15)

- **深模块**: `ao_reference_graph_db.py` 以 `search()`、`neighbors()`、`explain()`、`diagnostics()` 四个只读接口封装 6173 个节点与 6835 条关系；GUI 不解析二进制，也不持有关系判断规则。
- **生成器**: `tools/build_reference_graph.py` 独立、带边界检查地解析本机三语 `t_name._dt`、有效 `as*.dat` 头部、预加载模型和动作入口表；不执行上游生成 Python，不反汇编或执行动作指令。
- **覆盖**: 36 个三语角色 ID、305 个存档怪物、333 个 `ms` 状态文件、276 个有效 `as`、1052 条自定义战技、4171 个非空动作入口；305/305 怪物与 36/36 角色资源记录均有可追溯身份节点。
- **GUI**: 独立于存档的“资源索引”页支持三语名称、十六进制代码、`ms/as` 文件和动作入口反向搜索；可按六类资源或异常筛选，关系详情显示双向引用、来源与可信度。空搜索最多填充 500 行，完整关系仍可通过搜索访问。
- **保守处理**: 7 组跨三语同名记录只建立 `possible_variant_of` 候选关系；112 条不能证明为当前本地 `as` 入口的战技引用保留为 `unresolved_action_references`，不声明为存档技能 ID，也不进入写入路径。
- **来源策略**: `data_cn` 的 18 个 `as` 覆盖脚本优先作为 CLE 来源，其余回退 Falcom 日文基础脚本；日文/英文各 276 个脚本完整可用。JSON 只保存相对来源路径和 SHA-256，不保存本机绝对路径。
- **性能与验证**: 5.4 MB 图首次加载约 123 ms，跨 6173 节点搜索约 2–4 ms；生成两次 SHA-256 完全一致。75 项单元测试通过，Windows Python 3.13 隐藏窗口验证 13 标签页、500 行上限、`ms72200.dat` 反向搜索及三语切换。
### 5. 宝箱进度、缺失清单与坐标提示 ✅ P0 完成 (2026-07-15)

- **旗标基址**: `0x1B008`；280 个宝箱使用相对偏移 `0x1E0 ~ 0x207` 加 0~7 位号读取。
- **数据来源**: Ouroboros/Falcom `ED7/EDAO/RecordViewer/bin/box.json` 提供旗标、场景文件、宝箱/触发坐标和触发范围。

- **GUI**: 独立只读宝箱页，显示总进度、缺失数量、地图、物品、取得状态、宝箱坐标和触发坐标；支持跨语言搜索、地图筛选和“仅显示缺失”。

- **样本验证**: 两个 NISA 样例存档均可按该旗标表解析为 117/280 已取得；仍建议后续用同一进度开箱前/后存档做单 bit 对照。
