# Ao NISA PC 队伍 ID 核验

## 结论

NISA PC 存档 `0x1AFE0~0x1AFEE` 的 8 个 u16 字段仍使用“逻辑队伍 ID”，并不直接使用 `t_name._dt` 的角色外观/战斗资源变体 ID。旧版主力角色顺序没有变化，但旧映射漏掉了 NISA 自然存档中出现的临时同行角色。

特别需要区分：

- 队伍槽 `5`：当前游戏中代表莉夏。
- `t_name._dt` 的 `0x0005`：银的早期战斗资源。
- `t_name._dt` 的 `0x0020`：莉夏后期战斗资源。

因此不能直接把 `t_name` ID 表整体用于队伍下拉框。

## 交叉证据

1. 本机 NISA 日/英/中三份 `t_name._dt` 对应一致：`0x0005=银/Yin`、`0x0020=莉夏/Rixia`，并列出其他角色资源 ID。
2. 最新自然存档的队伍槽是 `[1, 2, 5, 0, 4, 3, 255, 255]`；实时游戏菜单显示六人为罗伊德、艾莉、缇欧、瓦吉，支援为兰迪、莉夏。集合完全对应 `{0,1,2,3,4,5}`，证明逻辑队伍槽中的 `5=莉夏`。
3. 对本机全部有效历史存档做只读统计，主力 ID `0~10` 均自然出现；此外出现临时同行 ID `61、82、140、160、161`。这些值能与 NISA `t_name` 的约纳、琪雅、莉丝修女、雷蒙德、秦一一对应。
4. `15=特殊魔兽` 同时存在于 NISA `t_name` 和旧 BZH 队伍代码表，但在本机历史存档中未自然出现，保留为有来源的特殊选项。

## 本机自然存档交叉验证（2026-07-15）

用户人工确认的 `data0003/0014/0027/0050/0059/0063/0081/0085` 队伍顺序，与 8 个队伍逻辑槽逐项一致。其中再次确认 `0x00A1` 为秦、`0x008C` 为莉丝修女，且前 4 槽为主队、后续槽为支援/同行角色。

角色外观区域则是 12 个固定角色槽，而不是当前队伍顺序：罗伊德、艾莉、缇欧、兰迪、瓦吉、莉夏、蔡特、亚里欧斯、诺艾尔、达德利、加尔西亚、其他。扫描 122 个校验有效的本机自然存档后，观察到瓦吉槽会由 `0x0004` 切换为后期资源 `0x001F`，莉夏槽会由银 `0x0005` 切换为莉夏 `0x0020`。间章湖水浴场的 `data0032~0037` 还稳定出现 `0x0060/0x0061/0x0062`，分别对应罗伊德、兰迪、瓦吉的泳装资源。
