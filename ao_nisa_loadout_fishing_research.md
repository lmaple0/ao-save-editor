# Ao NISA 人物装备、回路与钓鱼手册研究

## 研究范围

- 目标格式：NISA PC `savedata.dat` 解压后的 `0x2643C` 字节主体。
- 证据：本机 124 个校验有效的自然存档、当前游戏装备画面、项目物品多语言索引，以及旧 BZH 物品分类表。
- 安全边界：装备与回路已开放事务写入；钓鱼手册仍保持只读研究状态。

## 人物装备：槽位结构已验证

装备表从 `0x0304` 开始，11 个固定角色记录，每条 `0x0A` 字节。每条记录是 5 个连续的小端 `u16` 物品 ID：

| 相对偏移 | 槽位 |
| ---: | --- |
| `+0x00` | 武器 |
| `+0x02` | 服装 |
| `+0x04` | 鞋 |
| `+0x06` | 饰品 1 |
| `+0x08` | 饰品 2 |

固定角色顺序是：罗伊德、艾莉、缇欧、兰迪、瓦吉、莉夏、加尔西亚、亚里欧斯、诺艾尔、达德利、蔡特。

最新存档中艾莉记录 `0x030E~0x0316` 为 `0407 05F5 065A 0056 0056`，与游戏画面显示的血色光辉、纯净礼裙、斯托雷加Ω、寂静之蓝、寂静之蓝逐项一致。该完整 5 项序列在存档中只出现一次。

装备限制不能只按普通角色武器分类硬套：

- `0x0009` 是自然存档用于未启用角色的通用武器占位值。
- 加尔西亚武器槽在自然存档中会使用罗伊德武器，后期还会出现 `0x0474`；当前物品索引把 `0x0474` 归在蔡特武器。两种情况都必须作为已验证例外保留。
- 服装、鞋和两个饰品槽与对应物品分类一致；同一饰品可合法装备两份。
- 装备物品会从背包序列中移出。未来写入必须以“卸下旧物品、装上新物品、同步背包数量”为一个事务，不能只覆写槽位 ID。

## 回路：槽位与强化等级已验证

回路表从 `0x0CDC` 开始，同样是 11 个固定角色记录，每条 `0x1C` 字节。每条记录包含 7 组：

```text
u16 回路物品 ID
u16 槽强化等级（0、1、2）
```

- 第 0 组是核心回路槽，只接受 `circuit_core`。
- 第 1~6 组是普通回路槽，只接受 `circuit_normal`。
- 124 个有效自然存档中的 7 组分类和强化等级范围均符合此结构。
- 核心回路在 11 人之间没有重复装备；写入层应继续禁止同一核心回路同时装备给多人。
- 当前证据只确认“槽强化等级”，没有确认核心回路自身的成长等级/经验字段。
- 与装备相同，已装备回路不在背包物品序列中；GUI 写入会同步所有权与数量。

`ao_character_loadout.py` 提供解析、类别校验、槽强化等级校验、核心回路重复诊断和原子替换。未知 ID 原样保留，不自动纠正。

## 钓鱼手册：尚未达到安全写入条件

已确认的只有：

- Ao 共有 31 个鱼类物品 ID（`0x015E~0x017C`），名称已有中、日、英本地化。
- `Project1.exe` 的静态字符串能确认零之轨迹旧程序存在 `GetFishData` / `SaveFishData`，但不能证明 Ao NISA 沿用其偏移。
- 对早期、阶段中期和末期自然存档做了连续数组、交错记录和料理手册邻域扫描，没有找到能同时解释“鱼种是否登记、最大尺寸、次数”的 31 条稳定结构。
- `0x0AFA` 附近看似尺寸的连续数值实际是战斗/物品 ID 表，已排除；`0x19C98~0x19CA0` 是三份料理手册镜像，也不是钓鱼记录。

因此钓鱼手册继续保持只读研究状态。下一步最可靠的方法是受控差分：复制同一存档，分别完成一次新鱼种首钓、同鱼种更大尺寸、同鱼种非纪录尺寸，并各保存一次。只有同时得到三组前后样本，才能把“登记位、最大尺寸、次数”拆开命名。

## GUI 写入实现

人物装备/回路页现已支持下拉修改，选择后立即以单人物事务写入内存：

1. 将该人物原装备和回路临时退回背包池。
2. 校验并扣除新的装备与回路。
3. 同时写入人物槽位和物品区；任一步失败则恢复修改前数据。
4. 最终仍需点击主工具栏“保存存档”，由既有流程创建 `.bak` 并重算校验和。

强制规则包括角色武器分类、服装/鞋/饰品分类、核心/普通回路槽类型、同人物普通回路不重复、核心回路不被多人同时装备、槽强化等级只能是 `0/1/2`，以及背包物品必须真实可用。`0x0009` 仅作为未启用人物占位武器保留，不参与背包转移。

`t_orb._dt` 已确认包含角色导力器拓扑及属性限定信息，但尚未完成它与所有特殊普通回路的属性映射。GUI 会明确提示：属性限定槽尚未建模，修改回路后应先在存档副本中测试。

钓鱼手册取得三组受控差分样本前不实现修改按钮。
