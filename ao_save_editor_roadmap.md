## 料理手册 ✅ 已完成 (2026-07-15)

- **偏移**: `0x19C98`、`0x19C9C`、`0x19CA0` 三份镜像 `u32` 位图。
- **格式**: bit 1~24 对应 24 道基础菜谱，bit 0 保留；全开掩码为 `0x01FFFFFE`。
- **写入安全**: 三份镜像同步更新，同时保留料理掩码以外的未知位。
- **名称映射**: 按 NISA/Falcom `t_cook._dt` 索引 1~24 映射到每组三件料理中的基础料理 ID，复用现有 `ao_item_i18n.json` 中英日名称。
- **GUI**: 新增料理手册标签页，每道菜一个复选框，支持全选/全不选和全局中文/英文/日文切换。
- **验证**: 120 份 Ao 存档的三镜像全部一致；9 个已知进度点的置位数与游戏内数量匹配；临时副本完成全开、保存、重新加载 roundtrip。

### 4. 怪物图鉴目录、诊断与逐条编辑 ✅ P0 深化完成 (2026-07-15)

- **偏移**: `0x1B370 ~ 0x1BCF7`，共 305 个固定槽 × 8 字节；`0x1BCF0` 是最后一条记录的起始地址。
- **记录格式**: `u32 code + u8 flag + u8 resistance + u8 stats + u8 get_item`。
- **已验证完整资料 payload**: `08 FE FF FF`。部分状态只报告原始分类，不解释尚未验证的子位语义。
- **参考目录**: `ao_monster_reference.json` 静态合并 `MONSTER_CODES`、Ouroboros/Falcom `t_dbmon.py` 身份层和本机 NISA 三语 `ms*.dat` 名称；305/305 覆盖，其中上游启用 283、注释待复核 22。
- **GUI**: 独立怪物图鉴页，支持按代码、CLE 中文/日文/NISA 英文怪物名、`ms*.dat`、等级、来源和地点搜索；名称与地点随全局语言切换；提供 75 个细分地点下拉筛选；显示未登记/部分资料/全资料；可多选设为全资料或未登记。
- **安全策略**: 未选择记录和未知代码原样保留；报告重复代码；容量不足则拒绝写入；保留快捷页一键全开。
- **本地化来源**: 三语怪物名由本机 `data_cn/battle/dat`、`data/battle/dat`、`data/battle_us/dat` 的 `ms*.dat` 直接读取；地点由场景 `MapIndex` 对照三个本机 `t_town._dt` 表。302 项直接来自场景扫描；`ms82004`、村正、十六夜三个 MapIndex 为 0 的事件战斗使用已知本地地点标签在本机地点表中解析。每条记录保留场景文件和来源类型。
- **可复现性**: `build_monster_reference.py --game-root <NISA游戏目录> --check` 检查生成文件是否与来源一致；`t_dbmon` 使用 AST/文本解析，不执行上游脚本。
- **待验证**: 游戏内逐条回读、22 条上游注释映射的实际可见性，以及 4 个 payload 字节的精细语义。

### 5. 宝箱进度、缺失清单与坐标提示 ✅ P0 完成 (2026-07-15)

- **旗标基址**: `0x1B008`；280 个宝箱使用相对偏移 `0x1E0 ~ 0x207` 加 0~7 位号读取。
- **数据来源**: Ouroboros/Falcom `ED7/EDAO/RecordViewer/bin/box.json` 提供旗标、场景文件、宝箱/触发坐标和触发范围。
- **NISA 对齐**: 93 个场景文件全部存在于本机 NISA 数据；读取场景头 `MapIndex`，再对照 `data_cn`、`data`、`data/text_us` 的三语 `t_town._dt`，覆盖 44 张地图。
- **GUI**: 独立只读宝箱页，显示总进度、缺失数量、地图、物品、取得状态、宝箱坐标和触发坐标；支持跨语言搜索、地图筛选和“仅显示缺失”。
- **安全策略**: 不提供宝箱旗标修改，不进入保存写回路径。
- **样本验证**: 两个 NISA 样例存档均可按该旗标表解析为 117/280 已取得；仍建议后续用同一进度开箱前/后存档做单 bit 对照。
- **后续可选**: 地图底图标点需要逐地图建立世界坐标到图片像素的投影/校准关系，不属于当前 P0。

## P2 · 可延后

### 5. 角色战技 / S 技修改

- **Fight Skill**: 11 人 × 0x1C 字节 (0xAFC~0xC44)
- **S-Break**: 11 人 × 0x08 字节 (0xC64~0xCD0)
- **格式**: u16 技能代码数组
- **工作量**: 中 — 需要建立 150+ 技能名数据库
- **价值**: 低 — 容易导致游戏崩溃

- **原理**: 根据目标 LV 反推需要的 EXP，写入存档
- **工作量**: 小 — 需要碧轨 EXP 曲线表
- **价值**: 中 — 曲线改 EXP 已经可行

### 7. 系统存档

- **文件**: `SYSTEM/savedata.dat`
- **内容**: 仅含成就数据 (0x504~0x508 校验, 0x4C4~0x4CA 成就位图)
- **工作量**: 小
- **价值**: 低 — 很少需要单独改系统存档
